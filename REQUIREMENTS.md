# 제품 로드맵 관리 대시보드 - 요구사항 정의서

## 1. 개요

### 1.1 목적
매년 연초에 제품 로드맵을 작성하고, 해당 로드맵의 진척 상황을 추적/관리하는 내부용 대시보드

### 1.2 타겟 사용자
- 내부 직원
- 관리자

---

## 2. 핵심 기능

### 2.1 로드맵 수립

기존 운영 중인 제품에 대해 연간 로드맵을 수립하는 기능

#### 수립 워크플로우

**Step 1. 항목 입력**
- 문제점 또는 신규 기능 등록
- 입력 정보: **제목**, **설명**만 (간단하게)

**Step 2. 태스크 세분화**
- 각 항목에 대해 마일스톤과 세부 태스크를 수동으로 작성
- 마일스톤 추가 → 각 마일스톤 하위에 태스크 추가

**Step 3. 일정 수립**
- 세분화된 항목에 일정 배정
- 분기/월별 배치
- 마감일 설정

#### 항목 유형

| 유형 | 설명 |
|------|------|
| **문제점 (Issue)** | 현재 제품의 버그, 성능 이슈, 사용성 문제 등 개선이 필요한 사항 |
| **신규 기능 (Feature)** | 새롭게 추가되어야 할 기능 |

#### 계층 구조

```
목표 (Goal) - 유형: 문제점 또는 신규 기능
  └── 마일스톤 (Milestone)
        └── 태스크 (Task)
```

#### 그룹핑 옵션

- **유형별**: 문제점 / 신규 기능
- **기간별**: 분기(Q1, Q2, Q3, Q4) / 월별
- **조직별**: 팀별 / 제품별

### 2.2 진척률 관리

- 각 항목에 진척률(%) 수동 입력
- 0% ~ 100% 범위

### 2.3 시각화

#### 간트 차트

- 타임라인 기반 막대 차트
- X축: 시간 (분기/월별)
- Y축: 목표/마일스톤/태스크
- 일정 기간을 막대로 표시
- 진척률에 따른 색상 또는 채움 표시

#### 진척률 대시보드

- 전체 로드맵 진척률 요약
- 팀별 진척률 현황
- 제품별 진척률 현황
- 유형별(문제점/신규 기능) 진척률 비교
- 진척률 차트 (원형, 막대 등)

### 2.4 인력 관리

#### 인력 유형

| 유형 | 설명 |
|------|------|
| **기존 인력** | 현재 팀에 소속된 인원 |
| **신규 인력** | 해당 연도에 추가 예정인 인원 |

#### 인력 정보

- 이름
- 역할 (예: PM, 개발자, 디자이너, QA 등)
- 소속 팀
- 입사 예정일 (신규 인력의 경우)

#### 태스크 배정

- 각 태스크에 담당 인력 배정 가능
- 인력별 담당 태스크 현황 조회

### 2.5 내보내기/리포트

- 로드맵 데이터 PDF 내보내기 기능

## 3. 데이터 모델 (초안)

### 3.1 Goal (목표)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | INTEGER | PK |
| type | TEXT | 유형 ('issue' 또는 'feature') |
| title | TEXT | 목표명 |
| description | TEXT | 설명 |
| year | INTEGER | 연도 |
| quarter | TEXT | 분기 (Q1, Q2, Q3, Q4, 또는 NULL) |
| team | TEXT | 담당 팀 |
| product | TEXT | 제품명 |
| progress | INTEGER | 진척률 (0-100) |
| created_at | DATETIME | 생성일 |
| updated_at | DATETIME | 수정일 |

### 3.2 Milestone (마일스톤)
| 필드 | 타입 | 설명 |
|------|------|------|
| id | INTEGER | PK |
| goal_id | INTEGER | FK -> Goal |
| title | TEXT | 마일스톤명 |
| description | TEXT | 설명 |
| due_date | DATE | 마감일 |
| progress | INTEGER | 진척률 (0-100) |
| created_at | DATETIME | 생성일 |
| updated_at | DATETIME | 수정일 |

### 3.3 Task (태스크)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | INTEGER | PK |
| milestone_id | INTEGER | FK -> Milestone |
| title | TEXT | 태스크명 |
| description | TEXT | 설명 |
| assignee_id | INTEGER | FK -> Member (담당자) |
| due_date | DATE | 마감일 |
| progress | INTEGER | 진척률 (0-100) |
| created_at | DATETIME | 생성일 |
| updated_at | DATETIME | 수정일 |

### 3.4 Member (인력)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | INTEGER | PK |
| name | TEXT | 이름 |
| role | TEXT | 역할 (PM, 개발자, 디자이너, QA 등) |
| team | TEXT | 소속 팀 |
| type | TEXT | 유형 ('existing' 또는 'new') |
| join_date | DATE | 입사일/입사 예정일 |
| year | INTEGER | 해당 연도 |
| created_at | DATETIME | 생성일 |
| updated_at | DATETIME | 수정일 |

## 5. 화면 구성

### 5.1 레이아웃 구조

**기본 상태 (Dashboard만 표시)**
```
┌─────────────────────────────────────────────────────┐
│                                      [PDF 내보내기] │
├─────────────────────────────────────────────────────┤
│                                                     │
│              Dashboard (전체 화면)                   │
│              - 간트 차트                            │
│              - 진척률 대시보드                       │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**편집 시 (Edit 패널 슬라이드)**
```
┌─────────────────────────────────────────────────────┐
│                                      [PDF 내보내기] │
├──────────────────────┬──────────────────────────────┤
│                      │                              │
│    Edit 패널         │      Dashboard              │
│    (슬라이드로 열림)  │      (실시간 반영)           │
│                      │                              │
│         [닫기]       │                              │
└──────────────────────┴──────────────────────────────┘
```

- 기본: Dashboard 전체 화면
- 항목 클릭 또는 추가 버튼 → Edit 패널이 옆에서 슬라이드로 열림
- Edit에서 수정 시 Dashboard에 실시간 반영
- 편집 완료 후 패널 닫기

### 5.2 Edit 모드 (입력/수정)

- 로드맵 항목 CRUD
  - 목표(문제점/신규 기능) 추가/수정/삭제
  - 마일스톤 추가/수정/삭제
  - 태스크 추가/수정/삭제
- 진척률 입력
- 일정(분기/월, 마감일) 설정
- 트리 형태로 계층 구조 표시

### 5.3 Dashboard 모드 (시각화)

- 간트 차트: 타임라인 기반 일정 시각화
- 진척률 대시보드: 전체/팀별/제품별 현황
- Edit 모드에서 변경 시 실시간 업데이트

### 5.4 내보내기

- 현재 로드맵 상태를 PDF로 내보내기
